# Core Domains

This page defines the top-level knowledge domains for a software company.

## Platform Engineering

Owns shared foundations used by product teams.

- CI/CD pipelines
- build tooling
- developer experience
- internal platform services

## Application Engineering

Owns customer-facing and internal applications.

- frontend and backend services
- [[API Design]] and integration
- performance and quality engineering

## Cloud & Infrastructure

Owns runtime infrastructure and environment architecture.

- cloud accounts and tenancy
- networking and compute
- infrastructure as code
- capacity and cost controls

## Security & Compliance

Owns security posture and compliance controls.

- identity and access management
- threat modeling
- vulnerability management
- audit and evidence processes

## SRE & Operations

Owns reliability outcomes and operational excellence.

- incident response
- service reliability engineering
- SLO/SLI governance
- [[Observability]] standards

## Data & AI

Owns data products and intelligence systems.

- data platforms and pipelines
- analytics engineering
- ML/LLM integration patterns
- governance for data quality

## Product & Delivery

Owns execution model and value delivery.

- roadmap and planning
- requirements and prioritization
- release orchestration
- stakeholder communication

## Governance Rules

- Each note should map to one primary domain and optionally one secondary domain.
- Domain owners maintain index notes and key references.
- Cross-domain notes must explicitly link to both owning domains.
